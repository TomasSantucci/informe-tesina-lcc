En este capítulo se presentan los experimentos realizados para evaluar BasaltSLAM y el análisis
de los resultados obtenidos. Primero se buscó dimensionar el impacto de las mejoras descritas
en la sección x para lo cual se realizó un estudio de ablación que compara Basalt original
con distintas versiones del sistema que incorporan las funcionalidades del mapa de distintas maneras. Luego se evaluó
el sistema propuesto contra ORB-SLAM3 y OKVIS2 como referencia del estado del arte. En todos estos experimentos, no solo
se evaluaron métricas de error de trayectoria, sino también se hizo un énfasis en la performance, para determinar si
el sistema propuesto cumple con las limitaciones impuestas para funcionar en tiempo real.

Algunos experimentos fueron realizados en un servidor, que cuenta con un procesador Intel Core i5-13600 13th Gen con 32 GB de RAM.
Mientras que, aquellos que por diferentes motivos no se pudieron ejecutar allí, fueron realizados en una computadora que cuenta con
un procesador Intel Core Ultra 9 285H con 32 GB de RAM.

Para llevarlos a cabo se utilizaron todas las secuencias de los siguientes conjuntos de datos creados para la evaluación
de sistemas SLAM: EuRoC, TUM-VI benchmark, y el Monado SLAM Dataset que es un dataset específico de XR que representa mejor
el uso esperado del sistema. A continuación se detallan las características de los mismos.

\subsubsection*{EuRoC}
El dataset EuRoC (European Robotics Challenge) MAV (Micro Aerial Vehicle) es uno de
los datasets más populares y utilizados en la comunidad de robótica y visión por computadora.
Consiste en 11 secuencias grabadas en tres entornos diferentes: una sala de máquinas (Machine Hall) y dos salas
de laboratorio de la Escuela Politécnica de Zurich (Vicon Room 1 y 2). En la Figura 5.1 se pueden observar estos
entornos. Cada secuencia incluye datos de ground truth con una precisión milimétrica y están
disponibles los datos de calibración tanto intrínseca como transformaciones extrínsecas entre los sensores.
Todas las secuencias fueron capturadas utilizando un helicóptero Asctec Firefly equipado con
una cámara estéreo de resolución WVGA (752 x 480 px) a 20 fps y una IMU modelo ADIS16448
a 200 Hz.

Eleven datasets are provided in total, ranging from slow flights under good visual conditions to dynamic flights
with motion blur and poor illumination, enabling researchers to thoroughly test and evaluate their algorithms.
All datasets contain raw sensor measurements, spatio-temporally aligned sensor data and ground truth, extrinsic
and intrinsic calibrations and datasets for custom calibrations.


\subsubsection*{TUM-VI benchmark}
El TUM-VI Benchmark (Technical University of Munich Visual-Inertial Benchmark) es
otro dataset de referencia para la comunidad de visión por computadora ya que el mismo incluye
secuencias grabadas en diversos entornos, tanto interiores como exteriores, proporcionando un
amplio espectro de condiciones y desafíos para los sistemas de odometría visual-inercial.


Cada secuencia fue capturada con un dispositivo sostenido con la mano (handheld ) lo cual lo
hace particularmente interesante para evaluar la localizaci´on de sistemas XR. Para grabarlo se
utiliz´o una c´amara est´ereo con 512 × 512 px de resoluci´on a 20 fps y una IMU de alta precisi´on
a 200 Hz, asegurando una correlaci´on precisa entre los datos visuales y inerciales. Adem´as, el
benchmark incluye trayectorias de referencia obtenidas mediante m´etodos de captura de mo-
vimiento, garantizando un ground truth de alta calidad para la evaluaci´on cuantitativa de los
algoritmos
